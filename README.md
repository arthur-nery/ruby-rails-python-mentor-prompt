# 📕 Ruby/Rails + Python Mentor Prompt

**Ruby/Rails + Python Mentor Prompt** is an original mentor prompt created to guide a practical learning cycle based on the **one-function-per-day method**.

The prompt was designed to teach programming progressively, starting with Ruby and Ruby on Rails fundamentals, while also introducing Python as a complementary tool for automation, APIs, data workflows, and AI-related scripts.

This repository documents the original prompt used as the foundation for my public learning journey, GitHub practice, and transition from Ruby/Rails studies into a Python-focused phase.

---

# 🧠 What Is This Prompt

This is a **technical mentor prompt**.

Its purpose is to make an AI act as a structured programming mentor, combining the roles of:

- teacher
- senior Ruby on Rails developer
- code reviewer
- portfolio mentor
- operational risk advisor
- technical interviewer

Instead of learning many concepts at once, the prompt enforces a simple rule:

> Study only **one function, method, command, or operation per day**.

This makes the learning process easier to document, practice, publish, and review over time.

# ⚙️ How It Works

The prompt works by turning the AI into a strict daily mentor.

The mentor must follow a simple learning routine:

### 1️⃣ Review

The mentor starts by reviewing the previous function, method, command, or operation.

---

### 2️⃣ Teach One New Function

The mentor teaches only one new concept for the day.

It must not teach multiple functions at once, even if the topic is large.

---

### 3️⃣ Apply in a Minimal Example

The concept is applied in a small, practical example that can be written in a notebook or tested in the terminal.

---

### 4️⃣ Connect to GitHub and Portfolio

Each lesson should help produce something that can be documented, committed, and eventually shown as part of a public learning portfolio.

---

# 🧭 Learning Method

The central rule of the prompt is:

> Teach **one and only one** function, method, command, or operation per day.

The daily format includes:

1. Review of yesterday's function
2. One new function for today
3. Minimal example
4. Real-world usage
5. One common mistake
6. One small exercise
7. GitHub or portfolio note
8. Tomorrow's function, without teaching it yet

This structure was created to reduce overload and make progress visible.

# 🧑‍💻 How to Use

The recommended usage flow is:

1. Create a dedicated ChatGPT Project, if available in your account.
2. Add the full mentor prompt as the project instructions.
3. Use the same project for every daily lesson.
4. Start each session by asking the mentor to continue the one-function-per-day flow.
5. Study only the function of the day.
6. Write the exercise and a short note.
7. Commit the result to GitHub.
8. Share the progress publicly when relevant.

Using a dedicated project helps keep the mentor behavior more consistent over time, reducing context loss and making the daily learning cycle easier to maintain.

If ChatGPT Projects are not available in your account, you can still use the prompt by pasting it into a new chat whenever you start a lesson.
